# RTL Advisor V2.1 Implementation Plan

## 1. Objective

V2.1 is a recovery iteration driven by the frozen V2 benchmark. It keeps the core safety-first product contract—pre-synthesis analysis, generated/open RTL only, deterministic recommendation decisions, and formal verification of emitted variants—while correcting the feature-domain mismatch, improving opportunity detection, adding physical evidence, and repairing benchmark metrics that were mathematically incompatible with safe abstention.

The existing V2 benchmark remains immutable. V2.1 uses versioned corpora, models, commands, and artifacts, and does not become the live default unless every promotion gate in this document passes.

## 2. Frozen V2 postmortem

Add:

```text
rtl-advisor benchmark diagnose-v2
```

It reads only the frozen V2 lock, run records, calibration data, and report. It writes:

```text
artifacts/benchmarks/v2/postmortem.json
artifacts/benchmarks/v2/postmortem.md
```

The postmortem must:

- Verify the frozen lock and source-artifact hashes before analysis.
- Preserve the original V2 report and promotion decision byte-for-byte.
- Report normalized rejection causes and their frequency.
- Quantify calibration/live feature drift by family and feature.
- Separate rule misses, model rejections, interval rejections, and out-of-domain rejections.
- Evaluate clearly labeled shadow counterfactuals for point estimates, conformal intervals, OOD policy, and the random-forest challenger.
- Report direction confusion by PPA metric.
- Recompute exact-best accuracy using measured-utility ties within `1e-6`.
- Explain why the old direction-coverage denominator was incompatible with safe recommendation coverage.

Shadow results are diagnostic only and cannot alter the V2 result.

## 3. OpenROAD physical-evidence gate

Extend the existing OpenROAD planner and parser with:

```text
rtl-advisor benchmark openroad-lock-v2
rtl-advisor benchmark openroad-run-v2
rtl-advisor benchmark openroad-report-v2
```

Use OpenROAD-flow-scripts commit:

```text
036d106273e66855cd5214d49518fd0f0df7de61
```

This commit matches the currently used Nangate45 Liberty source. The lock records the ORFS commit, immutable Docker image ID, platform and plan hashes, Docker version, host architecture, and relevant configuration hashes.

Run 27 cases—three per optimization family—and all four equivalent variants, for 108 runs total. Runs use:

- Out-of-tree `DESIGN_CONFIG` files.
- Isolated `WORK_HOME` and stable `FLOW_VARIANT` values.
- The ORFS `finish` and `metadata` targets.
- Two workers by default.
- Resumable execution.
- No automatic retries.
- Explicit, audited `--retry-failed` for a manual retry pass.

A physical run is usable only when routing finishes, metrics parse successfully, and DRC violations equal zero.

The physical-evidence gate passes only when:

- At least 24 of 27 cases have complete four-variant result sets.
- At least two complete result sets exist for every family.
- Candidate-action agreement between synthesis and physical labels is at least 80%.
- Direction agreement is at least 75% separately for delay, area, and cell count.

If this gate fails, stop before V2.1 blind synthesis and expand or repair the physical labels. A failed gate may not be bypassed by retuning the blind benchmark.

## 4. V2.1 pre-synthesis feature contract

V2.1 introduces a new feature schema and leaves V2 unchanged.

- Use the same pre-mapping frontend for calibration, training, and live analysis.
- Extract model features from `kernel_top` only.
- Use `wrapper_top` only for synthesis and physical labels.
- Merge PySlang syntax facts with Yosys graph facts so source-level patterns such as equality-to-zero remain visible after Yosys normalization.
- Version and hash the feature order, types, normalization statistics, and topology signature.

Replace V2's hard all-feature min/max OOD rule with a family-specific nearest-neighbor distance:

1. Keep varying, non-categorical kernel features for each family.
2. Normalize each feature using its calibration median and IQR.
3. Compute leave-one-topology-out nearest-neighbor distances.
4. Set the family OOD threshold to the 95th percentile of those distances.
5. Record the nearest calibration topology, distance, threshold, and contributing features in every decision.

## 5. Rules and model stack

Improve registered rules using syntax facts and explicitly cover the V2 equality-to-zero comparator miss.

Train the following V2.1 models with seed `20260715`:

- Three `RandomForestRegressor` models with 500 trees for delay, area, and cell-count deltas.
- Three balanced random-forest direction classifiers with labels `improve`, `neutral`, and `degrade`; neutral means a measured delta within ±1%.
- One random-forest eligibility classifier for whether a candidate clears the balanced-profile measured gate.

All training predictions and threshold selection use grouped out-of-fold evaluation by topology signature. Refit final deployable models only after the calibration policy is locked.

Choose the risk threshold that maximizes opportunity coverage subject to all of:

- Calibration harmful recommendation rate at most 5%.
- Abstention specificity at least 90%.
- Balanced actionable accuracy at least 70%.

Rank eligible candidates using balanced utility. Emit a direction only when its classifier confidence clears the calibrated direction threshold; otherwise report it as uncertain.

Codex remains explanation-only. If explanation generation times out or fails schema validation, preserve the deterministic decision and score and set `explanation_status: failed`.

## 6. V2.1 corpus and ground truth

Use deterministic seed `20260715` and create:

```text
corpus/calibration-v21
corpus/heldout-v21
```

Generate 64 new calibration cases and eight new held-out cases per family:

- 576 new calibration cases.
- 72 new blind cases.
- 648 new topology signatures, all disjoint from V2.

Expand sparse domains:

- Adder operands: `{4, 5, 6, 7, 8, 10, 12, 16}`.
- Adder widths: `{8, 10, 12, 14, 16, 20, 24, 28, 32}` with operand-width product no greater than 128.
- Priority requests: `{4, 6, 8, 10, 12, 16, 20}`.
- Priority data widths: `{1, 4, 8, 12, 16, 24, 32}`.

Before any V2.1 synthesis:

1. Generate and freeze descriptors and RTL hashes.
2. Choose held-out cases using topology coverage plus an opportunity-propensity proxy trained only from V2 calibration data.
3. Verify all V2.1 topology signatures are disjoint from V2.

Each case contains `v0`, `v1`, `v2`, `v3`, and inequivalent control `n0`. Require lint and formal equivalence for `v1`–`v3`, and require formal rejection of `n0`.

Recompute V2 kernel features for the existing 360 calibration cases, combine them with the 576 new calibration cases, and train on that combined set. Never train, select thresholds, or modify rules using V2 blind labels or V2.1 blind labels.

Run synthesis for calibration only. For V2.1 blind cases, run lint and formal prevalidation, freeze a benchmark lock, and only then unseal synthesis.

## 7. Benchmark arms and execution

Evaluate six locked arms on the untouched V2.1 blind suite:

1. V1 registered rules.
2. V1 Codex advisor.
3. V1 hybrid.
4. V2.1 random-forest point predictor.
5. V2.1 risk gate.
6. V2.1 safe advisor.

Preserve the V2 execution scale and protocol:

- 480 benchmark records.
- 264 GPT-5.6 Sol xhigh calls.
- No cached substitutions for failed model calls.
- Execute outside a nested sandbox initially.
- Record every command, timeout, hash, and retry.

## 8. Corrected metric contracts

Balanced actionable accuracy is:

```text
(opportunity recall + abstention specificity) / 2
```

Direction coverage uses the number of recommended candidate metric slots as its denominator, rather than every case in the suite.

For each case, `best_candidate_ids` contains every candidate whose measured utility is within `1e-6` of the maximum. Exact-best succeeds when the selected candidate belongs to this set.

Ranking regret is conditional on covered, eligible opportunities. Missed opportunities are reported separately and are not disguised as ranking error.

Write new artifacts under:

```text
artifacts/models/v21
artifacts/benchmarks/v21
```

All new commands and schemas are explicitly versioned. The current live default remains unchanged until promotion.

## 9. V2.1 promotion gates

V2.1 is promoted only if all gates pass on the locked blind run:

- Micro balanced actionable accuracy at least 70%.
- Macro balanced actionable accuracy at least 65%.
- Opportunity coverage at least 50%.
- Abstention specificity at least 90%.
- Harmful recommendation rate at most 10%.
- At least a 10 percentage-point improvement over V1 hybrid, with the paired 95% bootstrap lower bound above zero.
- Direction accuracy at least 70% at conditional direction coverage of at least 90%.
- Direction accuracy at least 60% separately for delay, area, and cell count.
- Tie-aware exact-best accuracy at least 60%.
- Conditional normalized ranking regret at most 0.10.
- Accepted-candidate lint and formal pass rate of 100%.
- Candidate emission yield at least 80%.
- Deterministic decision and candidate stability of 100%.
- Advisor schema and reliability success at least 98%.
- Advisor semantic stability at least 85%.
- The OpenROAD physical-evidence gate passes.

Failure of any blind gate is reported without post-blind retuning. Any correction requires a new version and a newly generated blind suite.

## 10. Implementation sequence

### Phase 1 — Diagnose and freeze

- Persist this plan.
- Implement and run the frozen V2 postmortem.
- Add regression tests that prove the original V2 artifacts remain unchanged.

### Phase 2 — Establish physical evidence

- Implement OpenROAD lock, runner, resume/retry audit, and report commands.
- Build and identify the pinned ORFS image.
- Run the 108-case physical pilot and enforce the physical-evidence gate.

### Phase 3 — Repair the feature and rule layer

- Add V2.1 kernel-only feature extraction.
- Merge PySlang syntax facts.
- Add family nearest-neighbor OOD scoring.
- Repair demonstrated rule misses.

### Phase 4 — Generate calibration evidence

- Generate, lint, and formally validate the new calibration corpus.
- Synthesize its equivalent variants.
- Recompute kernel features for the old calibration corpus.
- Train grouped-OOF V2.1 regressors, classifiers, thresholds, and calibration reports.

### Phase 5 — Lock and run the V2.1 benchmark

- Generate the disjoint blind descriptors.
- Lint and formally prevalidate them.
- Freeze the benchmark lock before synthesis labels are available to advisors.
- Run all six benchmark arms and the fixed Codex call plan.
- Produce the promotion report without retuning.

### Phase 6 — Promote or retain the current default

- Promote V2.1 only if every promotion gate passes.
- Otherwise retain the current default, publish exact failure evidence, and scope the next version from the frozen report.

## 11. Operating boundaries

- Use only generated or explicitly approved open RTL.
- Never use company or proprietary RTL.
- Never modify source RTL in place.
- Never allow Codex to override the deterministic safety decision.
- Never use post-synthesis or physical labels as live inference features.
- Never tune against either blind suite after its lock is created.
- Pause for user intervention only when an external prerequisite, such as the Docker daemon, cannot be started safely from the terminal.
