# RTL Advisor v1: Codex-Assisted Pre-Synthesis Feedback

## Summary

Build a greenfield Python CLI that generates authorized SystemVerilog RTL and equivalent variants, extracts structural evidence with Yosys, obtains delay/area ground truth from Yosys/ABC, and benchmarks rules-only, Codex-only, and hybrid feedback.

- Use no company or third-party proprietary RTL.
- Default interactive Codex to GPT-5.6 Sol/xhigh.
- Benchmark both Sol/xhigh and Sol/ultra.
- Provide terminal and JSON output.
- Default to read-only recommendations; generate patches only when explicitly requested.
- Defer OpenROAD physical validation, ML training, IDE integration, power, and congestion analysis.

## Implementation

- Create a Python 3.13 package managed by `uv`, with code under `src/rtl_advisor`, generated cases under `corpus`, and disposable results under an ignored `artifacts` directory.
- Expose these commands:
  - `rtl-advisor setup`: verify Yosys, Verilator, Codex, and download the pinned Liberty file.
  - `rtl-advisor corpus generate`: deterministically generate development and held-out RTL cases.
  - `rtl-advisor analyze FILE --top TOP --mode rules|codex|hybrid [--effort xhigh|ultra] [--emit-patch]`.
  - `rtl-advisor synth CASE`: compile, map, and record delay/area evidence.
  - `rtl-advisor benchmark run --suite pilot`: run the blinded comparison.
  - `rtl-advisor benchmark report`: regenerate terminal and JSON summaries from cached raw records.
- Convert synthesizable SystemVerilog into Yosys JSON and build an operator/dependency graph retaining `src` attributes for source locations.
- Implement initial rules for mux depth, priority chains, serial arithmetic, mux placement around expensive operations, width growth, wide variable shifts, repeated decoding, and high fanout.
- Invoke `codex exec` ephemerally and read-only with a fixed prompt and JSON output schema. Codex-only receives RTL and the transformation catalog; hybrid additionally receives structural findings. Neither receives synthesis labels.
- Define findings with: category, source location, evidence, confidence, recommendation, registered transformation ID, predicted delay/area direction, and optional patch.
- Generate patches into a temporary copy only. Accept them only after Verilator lint, Yosys equivalence, and synthesis; never modify the source RTL automatically.

## Corpus And Ground Truth

- Generate 32 development cases and 36 opaque held-out cases across nine families: arithmetic resource sharing, adder/reduction association, mux placement, priority selection, decode factoring, comparator selection, variable shifting, width/signedness handling, and popcount/saturation structures.
- Give every case one baseline and three fixed-latency, functionally equivalent candidates. Use single-stage registered wrappers while verifying the underlying combinational kernels.
- Use disjoint parameters and seeds between development and held-out cases. Remove descriptive filenames, comments, and metadata from model-visible inputs to prevent label leakage.
- Require every variant to pass Verilator lint and Yosys formal equivalence before inclusion. Failed cases are excluded and regenerated from the next deterministic seed.
- Map every valid variant using local Yosys 0.63 and ABC against the Nangate45 typical Liberty file from OpenROAD Flow Scripts release `26Q2`, commit `036d106273e66855cd5214d49518fd0f0df7de61`. Store the upstream Apache-2.0 license, source URL, commit, and downloaded-file checksum in a lock manifest. The platform and licensing source are documented by [OpenROAD Flow Scripts](https://github.com/the-openroad-project/openroad-flow-scripts).
- Record mapped critical delay, cell area, cell count, tool versions, commands, seed, and logs. Cache results by hashes of RTL, constraints, library, and tool configuration.

## Benchmark And Decision Rules

Run five blinded arms:

1. Rules-only
2. Codex-only Sol/xhigh
3. Codex-only Sol/ultra
4. Hybrid Sol/xhigh
5. Hybrid Sol/ultra

- Run a four-case smoke test first, then one run across all 36 held-out cases.
- Repeat 12 stratified cases twice more for each Codex arm, producing 240 planned top-level Codex runs including the initial runs. Cache completed runs and allow one retry only for infrastructure or schema failures.
- Classify a recommendation as beneficial when it improves delay by at least 3% with no more than 10% area regression, or improves area by at least 5% with no more than 2% delay regression. Treat equivalence or compilation failure as harmful.
- Report top-recommendation actionable accuracy, beneficial/neutral/harmful direction accuracy, ranking regret against the best available candidate, patch lint/equivalence success, latency, model usage, and run-to-run agreement.
- Use bootstrap confidence intervals. Call hybrid/xhigh materially better only if it beats rules-only by at least eight percentage points and Codex-only/xhigh by at least five points, with the 95% interval for each improvement excluding zero.
- Report ultra as worthwhile only if it improves actionable accuracy by at least three points over xhigh. Keep xhigh as the live default regardless; reserve ultra for offline benchmarking and difficult audits.
- Preserve every raw result so the report is reproducible and unsuccessful hypotheses remain visible.

## Tests And Defaults

- Unit-test graph construction, source mapping, every rule, schemas, metric calculations, cache keys, and CLI failures.
- Golden-test Yosys/ABC parsing and deterministic corpus generation.
- Integration-test lint, equivalence, synthesis, mocked Codex responses, invalid JSON, timeout recovery, and patch rejection.
- Acceptance requires all 36 held-out cases to have valid ground truth, every benchmark arm to produce schema-valid results or an explicit failure record, and the report to rebuild solely from stored artifacts.
- Change the live Codex defaults to `model = "gpt-5.6-sol"`, `model_reasoning_effort = "xhigh"`, and `plan_mode_reasoning_effort = "xhigh"`. Benchmark commands override effort explicitly to `xhigh` or `ultra`; the installed Codex model catalog confirms both levels for Sol, while the [official model page](https://developers.openai.com/api/docs/models/gpt-5.6-sol) identifies Sol as the frontier GPT-5.6 model.
- OpenROAD is a later validation backend. Its interface should be anticipated behind the synthesis backend abstraction, but Docker startup and physical-design execution are not part of v1.
